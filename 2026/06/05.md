## Headroom

https://qiita.com/shinkai_/items/61b10d10c63db47a64e7

- Netflixのエンジニアが開発したClaude等のAIエージェントのトークン数を減らすためのツール
- JSON、ログなどの大量のデータを読み込ませたいときに有効
- ソースコードから設計書を生成する、設計書からコードを生成するといったコード中心のワークフローでは効果は限定的とのこと

SREだとログ調査とかも多いので良さそう。

### 仕組み（ざっくり）

Claude Code → headroom（手元のPC内）→ 圧縮 → Anthropic API

- やっていること：送信を横取り → 種類を見分けて圧縮 → Anthropic に転送
- `headroom wrap claude` のように包むと、Claude Code の通信が自動でこのプロキシ経由になる（コード改変不要）
- 元データは手元に残るので、必要なら原文を取り戻せる（外に出るのは圧縮版だけ）

つまり「Claude Code と Anthropic のあいだに立って、送るテキストをこっそり短くしてから渡す関所」というイメージ
