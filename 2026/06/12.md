## Claude Code 最近の機能メモ

### 1. `/goal` — 完了条件を満たすまで自走

1往復ごとに「続けて」と言う必要がなく、**測定可能なゴール**を渡すと、達成判定 → 修正を自動で繰り返す（裏側で小型 AI が達成判定）。

```bash
# テストが全件パスするまでリファクタリング
/goal 「npm test」がエラーなしで全件パスするまでリファクタリングを継続し、終わったら変更点を要約して。

# エラー原因の特定〜修正まで一気通貫
/goal ログにあるNullPointerのエラー原因を特定し、修正が完了するまで実行して。
```

### 2. 並行タスク管理 — `/bg` + `claude agents`

重いタスクをバックグラウンドに逃がし、**Agent View** で進捗・ユーザー操作待ちの状態を一覧・切り替えできる。

```bash
/bg              # 重いタスクを BG に回してセッションをデタッチ
claude agents    # 並行実行中のエージェント状態を一覧
```

### 3. レビュー・計画コマンド

GitHub 連携 + 深い思考モデル（Opus 4.7 等）で、レビューや実装計画の精度が上がる。

```bash
/ultrareview   # 差分・PR をセキュリティ・パフォーマンス観点含めて詳細レビュー
/ultraplan     # 新機能・新アプリのステップごとの実装計画
/sof-pr        # PR 作成後、CI エラーやレビュー指摘をクラウド上で自動修正
```

### 4. トークン・コスト管理 — `/usage` `/priority`

直近 5 時間の利用制限（API 枠の達成率）やトークン消費の内訳をターミナルから確認できる。

```bash
/usage      # トークン消費量・利用制限までの猶予（%）
/priority   # 優先度・トークン消費の詳細追跡
```

### 5. ノーフリッカーモード — CLI のちらつき抑制

CLI の描画リフレッシュによる画面のチカチカを抑える。

```bash
/tu fullscreen
```

または `~/.claude/settings.json` に設定:

```json
{
  "interactivity": {
    "noFlicker": true
  }
}
```

## Cloudflare、従業員やアプリごとにAIの利用上限額を設定できるCloudflare AI Gatewayの新機能を発表

- https://www.publickey1.jp/blog/26/cloudflareaicloudflare_ai_gateway.html
- https://blog.cloudflare.com/ai-gateway-spend-limits/

### Cloudflare AI Gateway とは

OpenAI・Anthropic・Google などの AI サービスとアプリ（またはユーザー）の間に挟むゲートウェイ。

- 任意の AI モデルへのルーティング
- リクエストのキャッシュ
- モデル別のリクエスト数・トークン使用量・エラーのログ
- レート制限

### 新機能：ユーザー / アプリごとの利用料上限（Spend Limits）

会社や部署で **API キーを共有** していると、誰がいくら使ったか通常は判別できない。今回の機能で **従業員やアプリ単位に月額の上限額** を設定できるようになった。

- AI モデルの価格に基づき **リアルタイムで利用料金を計算**
- 上限は **AI モデル・プロバイダー・カスタム属性** ごとに予算を設定可能
- 上限到達時の挙動を選択できる
    - そのユーザー / アプリの AI サービスへのアクセスを **ブロック**
    - **安価な AI サービスへダウングレード**

Cloudflare 自身も社内の AI 支出管理に AI Gateway を利用しているとのこと。現時点では **クローズドベータ**（参加者募集中）。
