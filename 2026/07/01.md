## Go言語: CGO

Go から C の関数を呼び出す仕組み。

- メリット: 既存の C ライブラリを再利用できる。
- デメリット: ビルドに **C コンパイラが必要**になり、**クロスコンパイルが難しく**、環境依存になる。

> Go の「どこでも簡単にビルドできる」強みを一部手放して、C の力を借りる仕組み。

---

## ⭐️CGO_ENABLED=0 について

CGO を無効にして **純 Go だけ**でビルドする指定。利点は 3 つ：

- **libc に依存しない静的バイナリ**になる（1 ファイルで動く）。
- **クロスコンパイルが簡単**（C ツールチェーン不要）。
- **`scratch` / `distroless` の極小 Docker イメージ**に載る。

> CGO の話の裏返し。C を切る＝どこでも動く小さなバイナリが手に入る。

## Slowloris（スローロリス）攻撃

「速さ」ではなく「**遅さで居座る**」タイプの DoS 攻撃。

- **仕組み**: 大量のコネクションを張り、リクエストを**わざと完成させない**。サーバは「まだ途中だ」と待ち続け、**接続枠が埋まって**正規ユーザが繋がらなくなる。
- **特徴**: 帯域も CPU もほぼ使わず**非力な 1 台でも攻撃でき**、中身は正しい HTTP なので**検知しにくい**。
- **対策**: **タイムアウトで切る**（Go なら `ReadHeaderTimeout` 等）。1 IP あたりの接続数制限や、前段の nginx / WAF も有効。

> 「遅さで居座る」攻撃だから、防御は「居座らせない＝タイムアウトで切る」。
