## ⭐️Linux: 物理CPU・コア・スレッド・論理CPUの違いまとめ

■ 1. 物理CPU（厨房の数）
・意味：パソコンの基盤に直接はまっている、銀色の「物理的なチップの数」。
・例え：店舗（厨房）の数。一般的なPCやCloud Runの1コンテナは「1物理CPU」。

■ 2. コア（料理人の人数）
・意味：チップ（物理CPU）の中に内蔵されている、実際に計算を行う「独立した頭脳の数」。
・例え：厨房の中にいる「料理人の人数」。4コアなら4人の料理人がいる。

■ 3. スレッド（料理人の手の数）
・意味：1つのコア（頭脳）が「同時に処理できる作業ラインの数」。
・例え：料理人が動かせる「手の数」。1人で2つの作業ライン（2スレッド）を同時に回せる。

■ 4. 論理CPU（全体の作業ラインの総数）
・意味：システムやクラウドが認識している「窓口（スレッド）の合計数」。
・例え：厨房全体で「同時に動いている手の総数」。クラウドの「vCPU」もこれと同じ。

▼ 具体例：「1物理CPU / 4コア / 各2スレッド」の場合
・物理CPUの数 ➔ 1つ （チップは1個）
・コアの数     ➔ 4つ （料理人は4人）
・論理CPUの数  ➔ 8つ （4人 × 2本の手 = 8つの作業ライン）

※Cloud RunやOSの設定画面で「CPU：8」や「cpu: "2"」と指定するものは、すべてこの「論理CPU（手が何本分あるか）」を指しています。

## ⭐️Linux: タイムスライス

- コンピュータのOSがCPUの実行時間を非常に短い時間（ミリ秒やマイクロ秒単位）に分割し、複数の処理（タスク）に順番に割り当てる仕組み
    - `1つの論理CPU上で同時に動けるプロセスは1つだけ`
    - 実行可能な複数のプロセスに、`タイムスライス`と呼ばれる単位で順番にCPUを使わせる

```
時間 ──────────────────────────────────────────────▶

論理CPU: │ P0 │ P1 │ P2 │ P0 │ P1 │ P2 │ P0 │ ...
         └─┬──┘
           └─ 1つの「タイムスライス」（例: 数ミリ秒）

・どの瞬間を切り取っても、動いているプロセスは常に1つだけ
・高速で切り替わるので、人間には「同時に動いている」ように見える（＝並行/concurrency）
```

## ⭐️Linux: コンテキストスイッチ

- 論理CPU上でプロセスが切り替わること

例えば、以下のようなプロセス0で動作するコードがある場合、

```
<!-- プロセス0で動作 -->
func sample():
    foo();
    bar();
```

fooの実行直後にbarが必ずしも動くとは限らない。

fooの実行直後にコンテキストスイッチが発生してCPUが実行するプロセスが切り替わった場合、fooの実行後、少し時間を置いてから（再度コンテキストスイッチが発生してプロセス0に切り替わったタイミングで）barが実行される。

```
論理CPU上の実行の流れ（時間は右方向）

  ┌─ プロセス0 ─┐         ┌─ 別プロセス ─┐        ┌─ プロセス0 ─┐
  │  foo() 実行 │         │  ... 実行中 ... │        │  bar() 実行 │
  └────────────┘         └───────────────┘        └────────────┘
                ▲                          ▲
                │                          │
         コンテキストスイッチ        コンテキストスイッチ
         （P0 → 別プロセス）         （別プロセス → P0）
                └──────── この間、P0 は待たされる ────────┘

・foo() の直後に必ず bar() が走るとは限らない
・切り替わった隙に「時間差」が生まれる
```

処理に時間がかかった場合に、「コンテキストスイッチが発生したかもしれない」という視点を持つのが大切。

## ⭐️Linux: CPU, コア, スレッドとCloud Run

前述の「厨房」の例えで、`物理CPU → コア → スレッド → 論理CPU` の関係を1枚の図で押さえる。

```
【 1物理CPU / 2コア / 各2スレッド の例 】

┌──────────── 物理CPU（厨房 = チップ1個）────────────┐
│                                                    │
│   ┌── コア0（料理人）──┐   ┌── コア1（料理人）──┐   │
│   │  手    手         │   │  手    手         │   │
│   │ (Th0) (Th1)       │   │ (Th2) (Th3)       │   │
│   └───────────────────┘   └───────────────────┘   │
│                                                    │
└────────────────────────────────────────────────────┘

  論理CPU の数 = 手（スレッド）の総数 = 4
                = OS からは「CPUが4つある」ように見える
                = vCPU 4 と同じ意味
```

- **物理CPU**：チップ1個（厨房） → PCやCloud Runの1コンテナは基本これが1つ
- **コア**：チップ内の頭脳（料理人）。「本当に同時に計算できる数」の上限を決める
- **スレッド**：1コアが同時に回せる作業ライン（手）。SMT/ハイパースレッディングで1コア＝2手など
- **論理CPU（= vCPU）**：手の総数。**OSがタスクを割り当てる単位**であり、`cpu` や `CPU数` の設定値はこれを指す

> ポイント：**設定画面の「CPU数」「cpu: "2"」は、コア数ではなく論理CPU（手＝スレッド）の数**。
> 「1物理CPU / 4コア / 各2スレッド」なら、指定できる論理CPUは最大 **8**。

### Cloud Run の `cpu: "2"`

- アプリに **「論理CPU（作業ライン）を2つ」** 割り当てる設定
    - ＝ 手が2本ある ＝ タイムスライスで回す「土俵」が2つある、というイメージ
- 「CPUが1つでコア数が2つ」と捉えても実用上は問題ないが、**正確には「論理CPU（vCPU）が2つ」**
    - 裏側（データセンター）の巨大な物理CPUから「2手分」を切り出して貸してくれている状態
    - アプリから見れば「論理CPUが2つあるマシン」に見えるので、そう理解しておけばよい

```
アプリ（OS）から見える世界          クラウドの裏側（実態）

┌──── 割り当て分 ────┐             ┌───── 巨大な物理CPU ─────┐
│ 論理CPU0  論理CPU1 │ ◀─ 切り出し ─ │  ▢▢ ▢▢ ▢▢ ▢▢ ▢▢ ...   │
│  (手1)    (手2)    │             │  （多コア・多スレッドの山）  │
└────────────────────┘             └─────────────────────────┘
        cpu: "2"
        = 論理CPU 2つ = vCPU 2
```

- 論理CPUが2つあるので、2つのプロセスを**本当に同時に**動かせる（＝並列 / parallelism）
    - 対して、1つの論理CPU上でタイムスライスによって切り替えながら動かすのは「見かけの同時実行」（＝並行 / concurrency）
    - `cpu: "2"` にする ＝ この**並列度（同時に走らせられる本数）を2に増やす**、ということ

## ⭐️SMT/ハイパースレッディング

- **SMT（Simultaneous Multi-Threading）**：1つの物理コアを、OSからは「複数の論理CPU」に見せる技術
    - Intel の実装名が **ハイパースレッディング（HT）**、AMD だと単に SMT と呼ぶ（中身はほぼ同じ考え方）
    - これが「1コア = 2スレッド」の正体。**コア数は増えていないのに、論理CPUの数だけ2倍になる**

### なぜそんなことができるのか

コアの中身は1枚岩ではなく、役割の違う部品の集まりになっている。

- **実行ユニット**（計算する部品：足し算、掛け算、メモリアクセスなど）＝ 料理人の**腕・コンロ**
- **アーキテクチャ状態**（レジスタ、プログラムカウンタなど「今どこまで実行したか」の情報）＝ 料理人が見ている**レシピと手元のメモ**

SMT は、**アーキテクチャ状態だけを2セット持たせて、実行ユニットは共有する**。

```
【 SMT なしのコア 】              【 SMT ありのコア 】

┌─── コア ────────┐             ┌─── コア ──────────────────┐
│ レシピ帳 ×1     │             │ レシピ帳 ×2 （スレッド0/1）│ ◀ ここだけ2セット
│                 │             │                            │
│ ┌ 実行ユニット ┐│             │ ┌────── 実行ユニット ─────┐│
│ │ 腕・コンロ   ││             │ │      腕・コンロ         ││ ◀ ここは共有（1セットのまま）
│ └──────────────┘│             │ └─────────────────────────┘│
└─────────────────┘             └────────────────────────────┘

OSからは「CPU 1個」          OSからは「CPU 2個」に見える
```

厨房の例えで言うと、**料理人を増やしたのではなく、料理人1人に「2つの注文伝票を同時に持たせた」**状態。
